## Análise Detalhada

### Falha no Global Setup

**Arquivo:** `server/src/routes/test.routes.ts` (endpoint `POST /seed`)  
**Causa:** O `reset-db` preserva `tipoalbums`, `stickers` e `secaos`. Porém, se a coleção `tipoalbums` estiver vazia (p. ex. após falha parcial anterior, restart de DB, ou limpeza manual), o `global-setup.ts` chama `/seed`. O endpoint verifica `tipoCount > 0`; como é 0, tenta `Sticker.insertMany(stickerDocs)` — mas stickers já existem, causando erro de chave duplicada que derruba o processo do servidor.

**Sintoma:** Servidor Node.js encerra com stack trace `MongoBulkWriteError: E11000 duplicate key error collection: ndab-meualbum.stickers index: number_1`. Nenhum teste chega a rodar.

**Fix:** No endpoint `/seed`, antes de inserir stickers e secoes, limpar estas coleções (`deleteMany({})`). Assim, quando TipoAlbum não existe (estado inconsistente), o seed parte do zero para os dados do catálogo.
