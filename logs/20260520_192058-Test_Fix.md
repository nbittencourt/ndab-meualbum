## Análise Detalhada

### Grupo 1 — `abrir-pacotinhos` (15 falhas)
**Arquivo:** `client/src/pages/AbrirPacotinhosPage.tsx`  
**Causa:** `useBlocker` do react-router-dom requer um *data router* (`createBrowserRouter`/`RouterProvider`). A aplicação usa `<BrowserRouter>`, que é um *legacy router* sem suporte a essa API. Na renderização, `useBlocker` lança um erro que derruba todo o componente, exibindo uma página creme em branco.  
**Fix:** Migrar `client/src/main.tsx` para `createBrowserRouter` + `RouterProvider`.

### Grupo 2 — `perfil` testes de senha (3 falhas: RN-P19, RN-P22a, RN-P22)
**Arquivo:** `tests/perfil/perfil.spec.ts` linhas 128, 138, 157  
**Causa:** `page.getByLabel('Nova senha')` usa correspondência parcial por padrão e corresponde tanto ao input "Nova senha" quanto ao "Confirmar nova senha". No modo estrito do Playwright, múltiplas correspondências causam erro.  
**Fix:** Usar `page.getByLabel('Nova senha', { exact: true })` nos 3 testes.

### Grupo 3 — `perfil` cooldown de email (1 falha: RN-P12)
**Arquivo:** `client/src/pages/ProfilePage.tsx`  
**Causa:** Após a 1ª submissão bem-sucedida de alteração de email, o perfil é atualizado e `emailPendente` é definido. O formulário é renderizado condicionalmente com `{!emailPendente && (<form>)}` e desaparece. O teste tenta preencher o formulário novamente para receber o erro de cooldown, mas o elemento não existe mais.  
**Fix:** Remover a condição `{!emailPendente && (...)}` que oculta o formulário. Exibir o bloco de "Aguardando confirmação" **acima** do formulário (sempre visível). Assim o usuário pode submeter novamente e receber o erro de cooldown do servidor.

### Grupo 4 — `perfil` exclusão com mensagem (1 falha: RN-P27)
**Arquivo:** `client/src/pages/ProfilePage.tsx`, `client/src/pages/LandingPage.tsx`  
**Causa:** Após exclusão de conta, `navigate('/')` é chamado sem passar estado. A landing page não tem lógica para exibir mensagem de conta excluída.  
**Fix:** 
1. Em `ProfilePage.tsx`: `navigate('/', { state: { contaExcluida: true } })`
2. Em `LandingPage.tsx`: ler `location.state?.contaExcluida` e exibir banner/mensagem.

### Grupo 5 — `colar-figurinhas` botão não encontrado (1 falha: RN-CF11)
**Arquivo:** `tests/support/helpers.ts`  
**Causa:** O fixture `request` do Playwright é isolado do browser context (`playwright.request.newContext()` — sem cookies compartilhados). Após `loginPorUI(page, ...)`, o `page` tem o cookie de autenticação, mas `request` não. Chamadas autenticadas via `request` (como `criarAlbum`) recebem 401 silenciosamente. O álbum não é criado, o componente exibe CF0 ("Nenhum álbum ativo"), e o botão "Colar figurinha não registrada" nunca aparece.  
**Fix:** Em `usuarioAtivo`, após o `loginPorUI`, fazer também login via `request.post('/api/v1/auth/login', ...)` para popular o cookie jar do fixture.
