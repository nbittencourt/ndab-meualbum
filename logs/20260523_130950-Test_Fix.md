## Análise da causa-raiz

### Contexto

O teste verifica que, após uma troca de senha, a sessão aberta em outro contexto (ctx2/page2) é invalidada e redirecionada para a tela de landing ao tentar acessar `/home`.

A invalidação de sessão no servidor funciona corretamente:
- `PATCH /profile/senha` incrementa `tokenVersao` no banco de dados e emite um novo JWT para a sessão corrente.
- O middleware `requireAuth` verifica se o `tokenVersao` do JWT corresponde ao do banco; em caso de divergência, retorna 401.
- Ao receber 401 em `/auth/me` (chamado com `skipAuthRedirect: true`), o `checkAuth` no authStore define `user: null`, e o `ProtectedRoute` redireciona para `/`.

### Problema identificado: race condition

O teste clica em "Alterar senha" e **imediatamente** navega `page2` para `/home` **sem aguardar** a conclusão da requisição PATCH ao servidor:

```ts
await page.getByRole('button', { name: /alterar senha/i }).click();

// ← sem espera: a requisição PATCH pode ainda estar em andamento
await page2.goto('/home');
```

O Playwright não aguarda automaticamente as requisições de rede disparadas por um clique. Se `page2.goto('/home')` executar antes do servidor concluir o PATCH (e incrementar `tokenVersao`), o JWT antigo de page2 ainda é válido e `/auth/me` retorna sucesso — fazendo o teste falhar de forma intermitente (ou, neste caso, consistentemente, dependendo da latência).

---

## Plano de correção

**Arquivo a editar:** `tests/perfil/perfil.spec.ts`  
**Linha:** após o clique em "Alterar senha" (linha 159), adicionar espera pelo toast de sucesso antes de navegar `page2`.

O `ProfilePage` exibe o toast `"Senha alterada com sucesso. Você foi desconectado dos outros dispositivos."` no handler `onSuccess` da mutation. Aguardar esse texto garante que o PATCH foi processado e o `tokenVersao` já foi incrementado no banco.

```ts
// ANTES
await page.getByRole('button', { name: /alterar senha/i }).click();

await page2.goto('/home');
await expect(page2).toHaveURL('/');

// DEPOIS
await page.getByRole('button', { name: /alterar senha/i }).click();
await expect(page.getByText(/senha alterada com sucesso/i)).toBeVisible();

await page2.goto('/home');
await expect(page2).toHaveURL('/');
```
