# Plano — Issues #29 e #30 (Tela AL1 / Figurinhas que Faltam)

## Contexto

Duas melhorias na tela de gerenciamento de álbum (**AL1** — [`AlbumManagePage.tsx`](../../client/src/pages/AlbumManagePage.tsx)):

- **#29 — Remover "Ver Álbum":** o botão "Ver álbum" na barra de ações da AL1 é redundante com a ação "Figurinhas que faltam" (ambos exibem a grade completa por status). Deve ser removido. Esse botão é o **único** ponto de entrada para a rota `/albums/:id/visualizar` ([`AlbumVisualizarPage.tsx`](../../client/src/pages/AlbumVisualizarPage.tsx)); por decisão do usuário, a página e a rota também serão removidas (sem código morto).

- **#30 — Imprimir "Figurinhas que faltam":** adicionar um botão **Imprimir** no topo do popup ([`ListaFigurinhasModal.tsx`](../../client/src/components/ListaFigurinhasModal.tsx)) que imprime **apenas o popup**, em **A4**, fiel ao layout HTML de referência [`Lista de Figurinhas v1.html`](../design_handoff/Lista%20de%20Figurinhas%20v1.html). Layout de colunas: **países em duas colunas**; **primeira seção** e **duas últimas seções** em coluna única. Além disso: diferenciar `colada` × `repetida` por **forma/padrão** (não por cor), pois em impressão monocromática verde e vermelho se confundem, e deixar claro quais figurinhas estão no "bolo de repetidas".

### Divergência registrada — resolvida neste plano
A spec [`spec_albums.md`](../spec_albums.md) (RN-AL19/AL22/AL30) descreve "Figurinhas que faltam" como **geração de PDF síncrona**. A implementação atual já divergiu para um **popup modal** (entregue em #26, com testes em [`albums.spec.ts`](../../tests/albums/albums.spec.ts) afirmando "sem download"). As issues #29/#30 confirmam essa direção. A impressão via `window.print()` cobre a saída em papel/PDF (PDF via "Salvar como PDF" do navegador). Conforme orientação do usuário, **este plano inclui a atualização das specs** para sincronizar a fonte da verdade com as decisões — ver a seção **"Sincronização da fonte da verdade (specs)"** abaixo.

### Mapeamento posicional das seções (confirmado no catálogo)
`tests/_seed/panini_wc2026_secoes.json` → seção 1 = "Página Inicial"; seções 2–49 = países; seções 50–51 = "FIFA World Cup History" + "Coca-Cola". Logo, a regra do #30 mapeia por posição na lista ordenada:
- `secoes[0]` → coluna única
- `secoes.slice(1, -2)` → duas colunas (países)
- `secoes.slice(-2)` → coluna única

## Issue #30 — Imprimir "Figurinhas que faltam"

Todo o trabalho concentra-se em [`ListaFigurinhasModal.tsx`](../../client/src/components/ListaFigurinhasModal.tsx) + uma folha de estilo de impressão dedicada.

### 1. Botão Imprimir no topo do popup
- Adicionar botão **Imprimir** no header do dialog (ao lado do botão Fechar, linhas ~143–152), com `aria-label="Imprimir lista de figurinhas"` e ícone/texto.
- `onClick={() => window.print()}`.
- Deve ficar **oculto na própria impressão** (classe `no-print`).

### 2. Render de impressão (layout A4 fiel ao HTML de referência)
- Renderizar, dentro do modal, um container de impressão dedicado `<div className="lista-print" aria-hidden="true">` com: cabeçalho (marca "Meu Album" + nome do álbum + data de geração via `new Date().toLocaleDateString('pt-BR')`), faixa de stats (coladas/faltantes/repetidas/%), as seções e a legenda. Portar os tokens visuais do HTML de referência (fontes Archivo Black / Geist Mono, faixa preta, borda vermelha).
- **Agrupamento por colunas** usando o mapeamento posicional acima:
  - `secoes[0]` e `secoes.slice(-2)` → blocos full-width (coluna única).
  - `secoes.slice(1, -2)` → dentro de um wrapper `.lista-print-paises` com **duas colunas** (CSS grid `grid-template-columns: repeat(2, 1fr)` ou `columns: 2`), cada seção com sua grade de células.
- Reutilizar a derivação de status existente `statusCell()` e os totais já calculados (`totalColadas`/`totalFaltantes`/`totalRepetidas`/`pct`).

### 3. Folha de estilo de impressão
Novo arquivo **`client/src/components/listaFigurinhasPrint.css`** importado pelo modal (não há `@media print` no projeto hoje; `client/src/index.css` é o CSS global). Conteúdo:
- `@page { size: A4 portrait; margin: 7mm; }`
- `.lista-print { display: none; }` na tela.
- `@media print`:
  - Ocultar o app e a versão de tela: `#root { display: none !important; }` e ocultar o overlay/dialog on-screen; `.lista-print { display: block !important; }`.
  - `print-color-adjust: exact; -webkit-print-color-adjust: exact;` no container.
  - `.no-print { display: none !important; }`.
  - Regras de coluna (2 colunas países / 1 coluna especiais) e `break-inside: avoid` nas seções para evitar quebra feia.

### 4. Diferenciação monocromática por **forma/padrão** (decisão do usuário)
No render de impressão (e refletido na legenda), distinguir os 3 status **sem depender de cor**:
- `colada` (c): preenchimento **sólido** (quadrado cheio).
- `repetida` (r): **cantos arredondados + padrão hachurado** (ex.: `repeating-linear-gradient`), tornando-a inconfundível mesmo em P&B — é o marcador do "bolo de repetidas".
- `faltante` (f): borda **tracejada**, fundo transparente (já distinto).
- Atualizar a **legenda do print** para descrever as formas e anotar que "repetida = você tem no seu bolo de repetidas (para troca)". As cores atuais (verde/vermelho/tracejado) permanecem na **tela**; a folha de impressão é que garante o contraste de forma/padrão.
