# Plano — Issue #39 (M4 — Compartilhar lista de faltantes)

Adicionar um botão para gerar um **link público** (UUID) que dá acesso, sem autenticação, à lista atual de figurinhas que faltam de um álbum — como a página de impressão. O link é **persistido** para reuso, evitando gerar tokens repetidos. É a primeira fatia entregável do item N2 (compartilhamento social).

> **Branch:** `claude/product-roadmap-update-43u4oj`
> **Milestone:** M4 — ver [`../ROADMAP.md`](../ROADMAP.md)

---

## Contexto

Da issue [#39](https://github.com/nbittencourt/ndab-meualbum/issues/39): ao clicar em "Compartilhar", o sistema gera um link único (UUID); quem recebe **não precisa de autenticação**. É exibida uma página, como a lista de figurinhas para impressão, com a lista atual de figurinhas. Se o usuário já gerou um link antes, o sistema **persiste e reutiliza** o token (sem regerar).

A lista de faltantes hoje vive no popup [`ListaFigurinhasModal.tsx`](../../client/src/components/ListaFigurinhasModal.tsx) (entregue em #26/#30, com impressão A4 via `window.print`), alimentado por `GET /albums/:id/figurinhas`. Não há, hoje, token de compartilhamento nem endpoint público.

## Diagnóstico

| Arquivo | Situação |
|---|---|
| [`server/src/models/Album.ts`](../../server/src/models/Album.ts) | Sem campo de token de compartilhamento. |
| [`server/src/models/User.ts`](../../server/src/models/User.ts) | Já tem `publicId` (6 chars, único/esparso) — referência para o padrão de identificador público; o compartilhamento aqui é **por álbum**, então o token vai no `Album`. |
| [`server/src/routes/albums.ts`](../../server/src/routes/albums.ts) | `GET /albums/:id/faltantes` e `GET /albums/:id/figurinhas` existem mas exigem auth. Sem rota pública. |
| [`client/src/components/ListaFigurinhasModal.tsx`](../../client/src/components/ListaFigurinhasModal.tsx) | Popup da lista + botão Imprimir; **sem** botão Compartilhar. |
| [`client/src/lib/api.ts`](../../client/src/lib/api.ts) | `albumsApi.faltantes/getFigurinhas` exigem auth; sem métodos de share. |

## Plano de implementação

**1. Modelo — token persistido (`server/src/models/Album.ts`)**
- Adicionar `shareToken?: string` (UUID v4, índice **único e esparso** — só álbuns compartilhados têm valor).
- Gerar com `crypto.randomUUID()` (Node nativo) — não adicionar dependência.

**2. Backend — endpoints (`server/src/routes/albums.ts`)**
- `POST /albums/:id/share` (autenticado, dono do álbum): se `shareToken` já existe, **retorna o existente**; senão gera, salva e retorna. Idempotente.
- `GET /public/share/:token` (**público**, sem `requireAuth`): resolve o álbum pelo `shareToken`, monta a lista de faltantes **reutilizando a mesma lógica** de `GET /albums/:id/figurinhas`/`faltantes` (extrair helper compartilhado para não duplicar). Retorna apenas dados necessários para a lista (figurinhas por seção + status), **sem PII** do dono.
- Registrar a rota pública fora do grupo autenticado (ver como `createApp()` monta os routers).
- (Opcional) `DELETE /albums/:id/share` para revogar o link — avaliar com o dono; por ora, fora de escopo se não pedido.

**3. Frontend — gerar e copiar link (`ListaFigurinhasModal.tsx` + `api.ts`)**
- Adicionar `albumsApi.gerarShareLink(id)` e `albumsApi.getListaPublica(token)` em [`client/src/lib/api.ts`](../../client/src/lib/api.ts) (este último **sem** credenciais).
- Botão "Compartilhar" no header do popup (ao lado de Imprimir/Fechar), com `aria-label`; ao clicar, chama o POST, monta a URL pública e copia para a área de transferência (com feedback/toast). Ocultar na impressão (`no-print`).

**4. Frontend — página pública read-only**
- Nova rota pública (ex.: `/compartilhado/:token`) e página que consome `getListaPublica(token)` e **reutiliza o layout** da lista de faltantes (mesmos blocos de seção e legenda do `ListaFigurinhasModal`, respeitando RN-AL19 e o layout de colunas RN-AL36). Sem chrome autenticado (sem bottom nav/menu). Tratar token inválido com mensagem amigável.

## Sincronização da fonte da verdade (specs)

Atualizar [`docs/spec_albums.md`](../spec_albums.md):
- Adicionar RN para **link de compartilhamento da lista de faltantes**: geração sob demanda, **persistência e reuso** do token (sem regerar), página pública sem auth, conteúdo reflete o estado atual (sem cache obrigatório).
- Registrar a decisão **LGPD**: o link expõe apenas a lista de faltantes do álbum, **sem dados pessoais** do dono; acesso por token não-adivinhável (UUID). Referência: [`docs/legal/lgpd_guia_sistemas.md`](../legal/lgpd_guia_sistemas.md). Considerar nota sobre revogação futura.
- **Arquivar** a versão atual da spec em `docs/_hist/AAAAMMDD/spec_albums.md` antes de editar (regra do CLAUDE.md).

## Riscos e observações

- **Privacidade:** garantir que o endpoint público não retorne PII (nome/e-mail do dono); expor só nome do álbum e a grade. Token UUID v4 é não-adivinhável; documentar.
- **Estado vivo vs. snapshot:** decidir se o link reflete o estado atual (recomendado, igual ao popup) ou um snapshot. Padrão: estado atual (sem cache), alinhado a RN-AL19.
- **Reuso de lógica:** extrair a montagem da lista em um helper compartilhado entre a rota autenticada e a pública, evitando divergência de layout/cálculo.
- **Roteamento público:** confirmar que o SW/PWA e o `requireAuth` não interceptam `/public/share/:token` nem `/compartilhado/:token`.
