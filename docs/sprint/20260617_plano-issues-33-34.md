## #33 — Ordenar figurinhas pelo número  ·  Pequeno

### Contexto

Dentro de cada seção, as figurinhas são atualmente ordenadas pelo campo `number` usando a ordenação padrão do MongoDB. Como o campo é do tipo `String`, a ordenação é lexicográfica: `BRA10` vem antes de `BRA9` porque `'1' < '9'` em comparação de texto. A issue pede ordenação pelo sufixo numérico (ex.: `BRA9 → BRA10 → BRA100`).

### Diagnóstico

| Arquivo | Linha | Código atual |
|---|---|---|
| `server/src/routes/albums.ts` | 220 | `Sticker.find({ secaoId: { $in: secaoIds } }).sort({ number: 1 }).lean()` |

O `.sort({ number: 1 })` do Mongoose delega ao MongoDB a ordenação como string. A correção deve acontecer **após** o `find`, em JavaScript, extraindo e comparando a parte numérica.

### Plano de implementação

**1. Extrator numérico (lógica isolada)**

Criar helper de ordenação testável:

```ts
// Extrai o sufixo numérico de um código de figurinha (ex.: "BRA10" → 10, "FWC3" → 3)
function numSufixo(numero: string): number {
  return parseInt(numero.replace(/^[A-Za-zÀ-ú\s]+/u, ''), 10) || 0;
}
```

**2. Alterar `server/src/routes/albums.ts`**

Substituir o `.sort({ number: 1 })` da query por um `.sort()` em memória após o `find`:

```ts
// antes
Sticker.find({ secaoId: { $in: secaoIds } }).sort({ number: 1 }).lean()

// depois
Sticker.find({ secaoId: { $in: secaoIds } }).lean()
  .then(arr => arr.sort((a, b) => numSufixo(a.number) - numSufixo(b.number)))
```

Ou equivalente com `await` + `sort` inline na desestruturação de `Promise.all`.

**3. Testes**

Não existe teste para ordenação de figurinhas. Seguindo TDD:

- **Vitest (unit):** testar `numSufixo` para casos esperados: prefixos alfabéticos simples (`BRA1`→1, `BRA10`→10), multi-char (`FWC`), números com zero (`FWC01`→1), fallback para string sem número (`ABC`→0).
- **Vitest (integration) opcional:** mockar `Sticker.find` retornando lista desordenada e verificar que a resposta da rota `GET /:id/figurinhas` retorna figurinhas na ordem numérica correta.
- **E2E:** não é necessário; a ordenação visual de lista longa é cobertura de apresentação, não regra de negócio crítica.

### Arquivos modificados

| Arquivo | Mudança |
|---|---|
| `server/src/routes/albums.ts` | Remover `.sort({ number: 1 })` do Mongoose; adicionar `.sort()` JS por sufixo numérico |
| `server/src/__tests__/albums-ordenacao.test.ts` (novo) | Testes Vitest para o helper e integração da rota |

### Riscos e observações

- O volume de figurinhas por seção é pequeno (dezenas); ordenação em memória tem custo desprezível.
- Prefixos com caracteres especiais (acentos, espaços) são tratados pelo regex `\s` — verificar se o catálogo real usa algum prefixo assim.
- A ordenação na tela AL2 (`AlbumViewPage`) usa os mesmos dados do endpoint `GET /:id/figurinhas` — a correção vale para ambas as telas automaticamente.

## #34 — Colagem rápida sempre ativa  ·  Médio

### Contexto

A feature de colagem rápida foi implementada na issue #24 com um botão toggle ("Ativar colagem rápida") como mecanismo de prevenção de acionamento acidental. A issue #34 remove essa restrição: o modo colagem passa a ser **sempre ativo**, tornando o toggle desnecessário.

Impacto:
- O botão "Colar" aparece em **todos** os cards faltantes/repetidos sem que o usuário precise ativar nada.
- O menu `⋮` (remoção de colagem) aparece em **todos** os cards colados.
- O botão "Ativar colagem rápida" é **removido** da barra de ações da AL1.

### Diagnóstico

| Arquivo | Linhas | Elemento |
|---|---|---|
| `client/src/pages/AlbumManagePage.tsx` | 617 | `const [modoColagem, setModoColagem] = useState(false)` |
| `client/src/pages/AlbumManagePage.tsx` | 812–831 | Botão toggle "Ativar colagem rápida" |
| `client/src/pages/AlbumManagePage.tsx` | 165, 177 | Prop `modoColagem` em `StickerCardAL1` |
| `client/src/pages/AlbumManagePage.tsx` | 270–291 | Botão "Colar →" (modo OFF): `{isRepetida && !modoColagem && ...}` |
| `client/src/pages/AlbumManagePage.tsx` | 295–317 | Botão "Colar" inline (modo ON): `{modoColagem && !isColada && ...}` |
| `client/src/pages/AlbumManagePage.tsx` | 320–380 | Menu `⋮` (modo ON): `{modoColagem && isColada && ...}` |

### Plano de implementação

**1. Remover o estado `modoColagem`**

- Excluir a linha `const [modoColagem, setModoColagem] = useState(false)` em `AlbumManagePage`.
- Remover o bloco `setMenuAbertoNumero(null)` do handler do toggle (a linha passa a não ter referência).
- O estado `menuAbertoNumero` e `removerNumero` permanecem — continuam sendo usados pelo menu `⋮`.

**2. Remover o botão toggle da barra de ações**

Excluir o bloco comentado `{/* Ativar colagem rápida — toggle */}` com o `<button>` das linhas 812–831.

**3. Simplificar `StickerCardAL1`**

- Remover a prop `modoColagem` da assinatura e do tipo `Props`.
- Remover o botão "Colar →" (modo OFF, linhas 270–291) — com colagem sempre ativa, a navegação para `/figurinhas` pelo card perde sentido.
- Alterar condição do botão "Colar" inline: `{modoColagem && !isColada && ...}` → `{!isColada && ...}`.
- Alterar condição do menu `⋮`: `{modoColagem && isColada && ...}` → `{isColada && ...}`.

**4. Remover `modoColagem` da cascata de props em `SecaoGrid`**

`SecaoGrid` recebe e repassa `modoColagem` para cada `StickerCardAL1`. Remover o parâmetro da assinatura e todos os usos.

**5. Atualizar testes E2E**

O arquivo `tests/figurinhas/colar-rapido-al1.spec.ts` testa o comportamento atual do toggle. Com a mudança de regra de negócio, os testes precisam ser reescritos antes da implementação (TDD).

Mapeamento de cada teste existente para o novo comportamento:

| Teste atual | Ação necessária |
|---|---|
| `modo desligado (padrão): cards sem botão Colar inline nem menu ⋮` | **Reescrever**: o estado "desligado" não existe mais. Substituir por: "cards de figurinha faltante exibem botão Colar sem ativação prévia". |
| `ativar modo exibe toggle com aria-pressed=true` | **Remover**: o toggle não existe mais. |
| `modo ativo: colar faltante sem estoque usa colagem direta` | **Simplificar**: remover o `click` no toggle; o botão Colar já deve estar visível. |
| `modo ativo: colar repetida com estoque debita o estoque` | **Simplificar**: remover o `click` no toggle. |
| `modo ativo: menu ⋮ em card colado → Remover exige código → remove colagem` | **Simplificar**: remover o `click` no toggle. |

**Regra anti-subversão:** as edições nos testes são permitidas por serem consequência direta de uma mudança intencional de UI (remoção do toggle), conforme o CLAUDE.md. As asserções de regra de negócio (colar, remover, confirmar com código) são **mantidas** — apenas os passos de ativação do toggle são removidos.

**6. Verificar typecheck e lint**

Após remover a prop `modoColagem`, rodar `npm run typecheck` para confirmar que não há referências residuais ao tipo.
